FieldNote — Quick Guide
What Is FieldNote?
FieldNote is a lightweight expedition journal for recording observations, findings, locations, hypotheses, and traces during research or exploration work. Everything is stored inside Akasha as structured semantic observations.
A FieldNote can be used for:
  • archaeology
  • field biology
  • qualitative research
  • historical investigation
  • worldbuilding
  • travel notes
  • speculative design
  • investigative writing
Think of it as a living research notebook connected to a semantic graph.
Basic Workflow
  1. Create a FieldNote
  2. Open it
  3. Add observations
  4. Review accumulated traces
Login
Enter your identity and your Akasha passphrase, then press Open the Field Book.
Creating a New FieldNote
At the top toolbar, enter a title and optionally add a project, region, and season. Then press New.

Opening Existing FieldNotes
The left sidebar contains all available FieldNotes. Click any entry to open it. The current FieldNote is highlighted.
Recording Observations
Use the lower input area. Fill in the fields, then press Add or use Ctrl+Enter (Windows/Linux) or Cmd+Enter (macOS/iPad).

Reading Observations
Press ↻ Read to reload the current FieldNote. Observations appear in chronological order. Different roles are color-coded.
Deleting a FieldNote
Open the FieldNote first, then press Delete. A confirmation dialog will appear. Deletion cannot be undone.
Export & Import
Export… — saves the current FieldNote (root + all observations + links) as an .akasha capsule file to your device. The button is visible only when a FieldNote is open.
Import… — opens a file picker. Select an .akasha capsule file to load its atoms into Akasha. Imported atoms are placed in an isolated pending scope until verified. The original FieldNote on the source device is not affected.
Notes About Confidence
Confidence is optional. This is not statistical certainty — it is a human field judgment.
ValueMeaning

0.8Strong hypothesis

Suggested Workflow
  1. Record raw observations quickly
  2. Separate observations from interpretation
  3. Add confidence later
  4. Re-read notes after accumulation
  5. Look for emerging patterns
FieldNote works best when used continuously over time.
Philosophy
FieldNote is intentionally lightweight. It is not trying to replace GIS systems, laboratory databases, publishing platforms, or institutional archives. Instead, it acts as a semantic field journal for living observations and evolving interpretation.
The goal is not only to store information. The goal is to preserve traces of thought while the investigation is still alive.
